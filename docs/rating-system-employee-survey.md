# 評分制度 — 員工意見調查｜Google 表單逐題稿（v2）

日期：2026-07-15
用途：讓同仁匿名填寫，評估「互評打分 → 連動時薪／分紅」這套制度的**好用度**與**可信度**。
交付形式：Google 表單。以下每題都標了**題型／選項／是否必填**，可直接照建。

> v2 調整（依使用者指示）：① 移除所有個人背景題（身分、年資）；② 封閉題一律採 1–5 分；
> ③ 開放題全部集中到最後 4 題；④ 填寫負擔改為中立、非負面引導的 1–5 題；
> ⑤ 報酬連動改為「時薪／分紅」——績效分數影響**計時的時薪**與**正職每季分紅（來自每季盈餘）**。

---

## 0. 建表前一定要先設定的事

1. **關閉「收集電子郵件地址」**。這份問卷成立的前提是匿名——它要挖的正是同仁不敢當面講的話（放水、不公平、主管調分）。一旦能對到人，就會得到跟現行互評一樣的「客氣分」。
2. **不要求登入 / 不限「僅回覆一次」**（限一次需登入，會破壞匿名）。
3. **不設任何姓名或身分欄**。想被單獨找去聊的人，請他私下直接找老闆。
4. **量表方向一律「左負右正」**：所有 1–5，都是 `1＝最不同意/最差`、`5＝最同意/最好`。分析時只有下面標 **⚠️反向** 的敘述例外（同意＝壞消息）。
5. 建議用分段（Section）切頁，手機一段一頁比較不累。

# 逐題稿

> 題型說明：多數題用 **1–5 單選方格（grid）**，同一組敘述共用 `非常不同意 → 非常同意` 五欄。
> **灰字（→ 給建表人）** 不要放進表單。標 **⚠️反向** 的列，表單上不要顯示標記，只在分析時記得「同意＝負面」。

## A. 整體感受

**A1. 整體而言，你覺得目前這套評分制度合不合理？**　〔線性刻度 1–5・必填〕
`1 = 很不合理` … `5 = 很合理`

## B. 介面與操作

**B. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 / 不同意 / 普通 / 同意 / 非常同意`
- 找到並打開評分頁面很容易
- 每一題我都看得懂在問什麼
- 實際點選、送出分數很順
- 在手機上填寫整體很流暢

## C. 填寫體驗（中立 1–5；不預設好壞）

**C. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- 這份問卷的填寫份量，我可以負荷
- 5 天的填寫窗口，時間很充裕
- 從第一位評到最後一位，我都能維持一樣的專注

> → 給建表人：這三題刻意用中性、可正可負的說法取代舊版「填到後面會亂點」。三題偏低＝負擔或疲勞影響資料品質，但問法不誘導。

## D. 評分題目本身

**D. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- 我被評到的項目，大多是我實際有在做的工作
- 幫我評分的人，清楚我平常真正在做什麼
- 21 個評分項目，對我的工作來說是合理的
- ⚠️反向　幫別人評分時，我常遇到「對方沒做過、我不知道怎麼給」的項目
- 遇到不熟的項目，我會盡量想清楚再給分

## E. 評分機制：互評與自評

**E. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- ⚠️反向　大家互評時，普遍傾向給高分、不太願意給低分
- 現在的分數，能真實區分出誰做得好、誰只是普通
- ⚠️反向　因為分數會影響對方的時薪／分紅，我給低分時會有壓力
- 自評和別人評的分數同樣重要，我覺得這樣公平
- ⚠️反向　這種算法下，敢給自己打高分的人比較占便宜
- 自評的份量應該再降低（或不算進正式分數）

> → 給建表人：第 1、3、5 列同意＝制度警訊；第 6 列同意＝支持調整自評權重。分析時分開看，勿加總。

## F. 主管調整分數

**F. 主管可以對態度分／表現分做加減調整。以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- 這件事對我來說是透明的（我知道、看得到、也知道原因）
- 我希望能看到自己被調整前／後的分數
- 我希望知道每次被調整的理由
- 我希望調整前後有人主動告知我

> → 給建表人：第 1 列偏低＝目前不透明；後三列偏高＝同仁要的是知情權。

## G. 時薪／分紅連動與信任

> 區段說明（貼在區段開頭）：這套分數會影響**計時的時薪**與**正職每季的分紅（來自每季盈餘）**。

**G. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- 我清楚自己的分數怎麼換算成時薪／分紅
- 我相信這套分數算出來的時薪／分紅是公平的
- 把工作做好，真的會反映在我的分數和時薪／分紅上
- 用同事互評來決定彼此的時薪／分紅，我覺得合理
- 知道分數會影響時薪／分紅，讓我更有動力把工作做好

## H. 回饋與申訴

**H. 以下說法，你同意嗎？**　〔單選方格・必填〕　欄：`非常不同意 → 非常同意`
- 分數出來後，有人會和我討論結果、幫我一起變好
- 我曾因為看到自己的分數，而實際調整了某個做法
- 對分數有疑問時，我知道可以找誰，也敢去找

## I. 整體報酬

**I1. 整體來說，你對現在的報酬（時薪／分紅，加上全勤、三節、年終）滿意嗎？**　〔線性刻度 1–5・必填〕
`1 = 很不滿意` … `5 = 很滿意`

## J. 最後：開放意見（都可留空）

**J1. 如果這套制度只能改一件事，你最想改的是什麼？**　〔段落・選填〕

**J2. 關於「分數出來之後」的回饋或申訴，你有什麼想法或希望？**　〔段落・選填〕

**J3. 關於薪資／分紅，你最希望店裡做的一件事是什麼？**　〔段落・選填〕

**J4. 還有什麼前面沒問到、但想讓老闆知道的？**　〔段落・選填〕

## 附錄三　分析時怎麼讀

1. **反向題單獨看**：D-列4、E-列1、E-列3、E-列5 這四項「同意＝壞消息」，不要跟正向題加總平均，會互相抵消。
2. **資料品質當前置條件**：先看 C 三題。若普遍偏低，代表負擔／疲勞在影響作答，所有分數要打折解讀。
3. **每格算「平均分＋低分比例」**：平均看整體，低分（1–2 分）比例看「有多少人明確不滿」——後者常比平均更早示警。
4. **開放題（J）逐則讀**：這 4 題是質性金礦，尤其 J2 最可能指出「下一版該補的功能」。
