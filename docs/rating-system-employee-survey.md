# 評分制度 — 員工意見調查｜Google 表單逐題稿

日期：2026-07-15
用途：讓同仁匿名填寫，評估「互評打分 → 連動時薪」這套制度的**好用度**與**可信度**。
交付形式：Google 表單。以下每題都標了**題型／選項／是否必填**，可直接照建。

---

## 0. 建表前一定要先設定的事

1. **關閉「收集電子郵件地址」**（表單設定 → 回覆）。這份問卷成立的前提是匿名——它要挖的正是同仁不敢當面講的話（放水、不公平、主管調分）。一旦能對到人，資料就會得到跟現行互評一樣的「客氣分」，整份問卷就白做了。
2. **不要求登入 / 不限制「僅回覆一次」**（限制回覆需登入，會破壞匿名）。寧可承擔極少數重複填答，也要保住匿名。
3. **不設任何姓名欄**。想被單獨找去聊的人，請他私下直接找老闆，不要放進表單。
4. 量表方向**全程一致**：所有「同意程度」一律 `非常不同意 → 非常同意`（左負右正）。分析時特別注意下面標「⚠️反向題」的列——那些是同意＝壞消息。
5. 建議用「分段（Section）」切成幾頁，手機一頁一段比較不累。若你想讓「正職／不參與互評的人」跳過互評題，可用 Q1 的答案設定「依答案跳到區段」（做法見文末）。

> • 「不知道／沒感覺」也是重要答案，照選沒關係。

## 區段六　評分機制：互評／自評／主管調整（信任構面 1、2、4——制度信任核心）

**Q14. 請就下列說法表達同意程度：**　〔單選方格 grid・必填〕
- 列：
  - ⚠️反向題　大家互評時，普遍傾向給高分、不太願意給低分
  - 現在的分數，能真實區分出誰做得好、誰只是普通
  - ⚠️反向題　因為分數會影響對方時薪，我給低分時會有壓力
  - 自評和別人評的分數同樣重要，我覺得這樣公平
  - ⚠️反向題　這種算法下，敢給自己打高分的人比較占便宜
- 欄：`非常不同意` / `不同意` / `普通` / `同意` / `非常同意`

> → 給建表人：⚠️反向題「同意＝壞消息」。第 1 列測**放水**、第 3 列測**人情壓力**、第 5 列測**謙虛吃虧**。第 2、4 列是正向題（同意＝好）。分析時分兩組看，別直接加總。表單上把「⚠️反向題」字樣拿掉，只留敘述。

**Q15. 如果可以改「自評」的份量，你希望？**　〔單選・必填〕
- 自評不要算進正式分數（只當參考）
- 自評份量調低
- 維持現在一樣重
- 自評份量調高
- 沒意見

**Q16. 你知道主管可以對你的態度分／表現分做加減調整嗎？**　〔單選・必填〕
- 知道，而且看得到自己被調了多少、為什麼
- 知道有這功能，但看不到自己被調的細節或理由
- 完全不知道有這回事

**Q17. 對於「主管調整分數」，你最希望的是？（可複選）**　〔核取方塊・必填〕
- 讓我看到自己被調整前／後的分數
- 讓我知道被調整的理由
- 事前或事後有人跟我說一聲
- 有機會表達不同意見
- 維持現狀就好
- 其他：＿＿（開放）

## C. 分析時怎麼讀（三個重點）

1. **反向題單獨看**：Q14 的三個 ⚠️ 列，同意比例越高＝制度問題越大。不要跟正向題加總成一個平均，會互相抵消。
2. **資料品質當前置條件**：先看 Q7／Q8。如果多數人說「填到後面亂點」，那所有分數結果都要打折解讀——這也直接回答了「這套分數到底有沒有意義」。
3. **交叉分析靠 Q1／Q2**：用身分（計時／正職／主管）和年資切開看。放水、人情壓力這種題，新人和老鳥、被評者和調分者的答案通常差很多。

## D.（選用）正職跳過互評題的設定方式

若正職不參與互評：Q1 設為「依答案跳到區段」，選「正職人員」→ 跳到「區段七」，略過區段四、五、六。計時人員與主管走完整流程。
