# 自動排班：只代入指休／指定上班（不生成班表）

日期：2026-06-24
檔案：`index.html`（自動排班彈窗）

## 目標

在自動排班彈窗加一個按鈕，只把每位員工勾選的指休／指定上班日期寫入班表，
**不跑** `autoGenAlgo` 的完整生成（不自動補休、不填班別、不滿足每日需求）。

## UI

彈窗底部、原「取消／生成班表」列上方，加一整列 outline 按鈕：
`📌 只代入指休/指定上班（不生成班表）`，onclick=`applyDesignatedOnly()`。

## 行為：`applyDesignatedOnly()`（同步語意）

逐位員工，依模式處理當月每一天（讀 `#leaves-{empId}` 內 `.selected` 按鈕）：

- **指定上班模式**（`designatedWorkMode`）：shift = `allowedShifts[0]||'C'`，
  allowedSet = allowedShifts ∪ {'C'}。
  - 勾選日：若該格已是 allowedSet 內班別 → 保留；否則寫 shift。
  - 取消勾選且原本是 allowedSet 內班別 → 清空（delete）。
- **指休模式**（其餘員工，含固定班規則員工）：
  - 勾選日：若不是 `指` → 寫 `指`。
  - 取消勾選且原本是 `指` → 清空（delete）。
- 其他日期（一般班別、休/特/國、固定班員工 disabled 的六日）→ 完全不動。

收尾：`saveSchedule` → 關彈窗 → `renderSchedule()` → toast 顯示更動格數。

## 與「生成班表」差別

生成填滿整月；只代入只寫勾選的指休/指定上班格，其餘留白。

## 驗證（preview 實測）

- 指休員工 `{1:指,2:指,10:A}`、改選 [2,5] → `{2:指,5:指,10:A}`
  （取消的清空、保留的留、新增寫指、工作班 A 不動）。
- 指定上班員工(E) `{1:E,2:E,20:休}`、改選 [1,25] → `{1:E,20:休,25:E}`
  （取消的清空、已是 E 的保留不覆蓋、新增寫 E、休 不動）。
