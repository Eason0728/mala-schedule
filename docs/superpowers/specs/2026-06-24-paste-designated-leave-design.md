# 自動排班彈窗：貼上文字帶入指定休日期

日期：2026-06-24
檔案：`index.html`（單檔，自動排班彈窗區塊）

## 目標

在「✨ 自動排班」彈窗中，讓使用者可以對「指定休模式」的員工貼上一段純數字日期文字，
點「套用」後自動把對應的日期按鈕勾選成指定休，省去逐天點擊。

## 範圍

- 只對「點選不能上班的日期」模式的員工列加輸入框（即 `!isWorkMode` 的員工，
  含一般指定休員工與固定班規則員工）。
- 「點選上班的日期」模式員工（`designatedWorkMode`）**不加**此功能。
- 此功能只改動彈窗內按鈕的勾選 UI 狀態，與現有逐天點擊行為一致；
  實際寫入班表仍由既有的「產生班表」按鈕負責，本功能不直接寫 localStorage。

## 互動流程

1. 在該員工列的輸入框貼上純數字日期文字，例如：
   - `9、10、15~17`
   - `9,10,15-17`
   - `9 10 15 16 17`
2. 點該列的「套用」按鈕。
3. 解析出日期清單後，**取代**該員工目前的選擇：
   - 先把該員工所有已選（`.selected`）的日期按鈕清空，
   - 再勾選文字中解析出的日期按鈕（沿用 `toggleLeaveDay` 既有的選中樣式）。
4. 套用後清空輸入框，toast 提示「✅ 已套用 N 天指定休」。
   若沒解析到任何有效日期，toast 提示「沒有解析到日期，請確認格式」，不清空輸入框。

## 解析規則：`parseDayListText(text, maxDay)`

- 回傳排序去重後的整數日期陣列。
- 分隔符：`、`、`,`、空白（含全形空白）可混用。
- 區段：`9~12` 或 `9-12` 展開為 9,10,11,12（含頭尾）；
  區段端點顛倒（`12~9`）視同 `9~12`。
- 過濾：`<1` 或 `>maxDay` 的日期直接忽略；無法解析的片段忽略，不中斷。
- 僅接受純數字（不處理 `M/D` 月份格式，因為已在當月彈窗內）。

## UI

- 位置：在 `empRow()` 內，員工名稱／hint 那行下方、日期按鈕格上方，新增一列控制項。
- 控制項：一個 `text` input（placeholder 範例 `9、10、15~17`）＋一個小「套用」按鈕。
- 樣式沿用既有 `.btn .btn-outline .btn-sm` 與表單 input 風格，寬度自適應、單列不換行。
- 套用邏輯透過 `applyLeaveText(empId, inputEl, maxDay)`：
  讀 `#leaves-{empId}` 容器內的 `.auto-day-btn`，
  先把現有 `.selected` 還原（套用 `toggleLeaveDay` 的「取消」分支樣式），
  再對解析出的日期對應按鈕套用「選中」樣式。
- **固定班規則員工**：六日的按鈕是 `disabled`（已自動休），
  套用時跳過 disabled 按鈕，不強制勾選，也不報錯。

## 邊界狀況

- 空字串／全是空白：toast 提示格式錯誤，不動現有選擇。
- 解析到的日期對應到 disabled 按鈕（固定班員工的六日）：略過該日。
- 重複日期（`9、9`）：去重後只勾一次。
- 日期超出當月（如 7 月貼 `32`）：忽略。

## 測試

沿用 SKILL.md 的無頭測試方式（Node mock localStorage/document）：
- `parseDayListText('9、10、15~17', 31)` → `[9,10,15,16,17]`
- `parseDayListText('9,10,15-17', 31)` → `[9,10,15,16,17]`
- `parseDayListText('9 10 32 0 -1', 31)` → `[9,10]`
- `parseDayListText('12~9', 31)` → `[9,10,11,12]`
- `parseDayListText('', 31)` → `[]`
